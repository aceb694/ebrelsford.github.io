code style

1. indentation

always indent the body of functions one level

function makeMapDoStuff(map) {
map.setCenter([0, 0]);
map.setZoom(4);
}
function makeMapDoStuff(map) {
    map.setCenter([0, 0]);
    map.setZoom(4);
}

same goes for if statements

if (allElseFails) {
giveUp();
}
if (allElseFails) {
    giveUp();
}

2. comments

they're like hints for your future self

style: function (feature) {
  var style = {
    fillColor: '#0f0f91',
    fillOpacity: 0.5,
    radius: 8,
    stroke: false
  };
  if (feature.properties.price > 4000) {
    style.fillColor = 'red';
  }
  return style;
}
style: function (feature) {
  // Style based on price
  var style = {
    fillColor: '#0f0f91',
    fillOpacity: 0.5,
    radius: 8,
    stroke: false
  };
  if (feature.properties.price > 4000) {
    style.fillColor = 'red';
  }
  return style;
}

you don't have to do one for every line

but aim to narrate your code, separating it into chunks

future self (and I) will thank you

3. blank lines

use one blank line between logically separate chunks of code

why make yourself scroll more than necessary?

4. variable names

longer is okay!

cx = ox - rd;
centerX = offsetX - radius;

you'll thank yourself when you have to read it later

and use camelCase

(start lowercase, but capitalize each word within the variable name)

var pointsgeojsonlayer;
var pointsGeoJsonLayer;
