In-class Excercises: Mapbox

Part 1: Mapbox, basic styles

  1. Create an account with Mapbox if you don't already have one.
  2. Log in to your Mapbox account.
  3. Once logged in, go to the Projects page and click the + New project button.
  4. Under the Style tab, select a style you like, then click Save.
  5. Go back to your Projects page.
  6. In another tab open your CartoDB account and pick a visualization you would like to add the base tiles to.
  7. In the visualization, click the Change basemap button, click the + next to "Yours", choose the Mapbox tab and enter the correct information in the fields given.

Part 2: Mapbox tiles in Leaflet

  1. Open a Leaflet map you have made in the past, either for in-class work or an assignment.
  2. Add the required CSS and JS files to your page to load Mapbox.js.

  3. Use L.mapbox.tileLayer to add your Mapbox layer from Part 1 to your map instead of the base layer you're currently using.

    As the documentation implies, there are some options here regarding which parameters to include. I recommend loading it this way: 

    L.mapbox.accessToken = 'your_access_token';
var layer = L.mapbox.tileLayer('your_map_id');

  4. Once you create the L.mapbox.tileLayer make sure you're adding it to the map using, for example: layer.addTo(map).

Part 3: Mapbox Studio

  1. Install Mapbox Studio on your computer if you haven't already, and open it.
  2. Pick a style to start from.
  3. Work through a number of CartoCSS properties in the style. Look at:
    • changing existing variables,
    • adding new variables,
    • styling features based on attributes,
    • styling features based on zoom level,
    • styling points,
    • styling lines,
    • styling polygons,
    • using patterns.
  4. Upload your style to mapbox.com.
  5. Add it to a CartoDB or Leaflet map.

Part 4: Mapbox examples

  1. Take a look at Mapbox's examples area. Many of them have less to do with Mapbox.js and more to do with Leaflet, so this can be a handy resource.
  2. Pick one of the examples, copy the code, and get it running on your machine or JSBin.
  3. Modify the example and play around with it.
