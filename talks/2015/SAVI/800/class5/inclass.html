In-class Excercises: Leaflet Plugins

Part 1: Styling Leaflet features

  1. Start with a Leaflet map (not made with CartoDB.js) from previous work.
  2. Using this jsbin as an example, style the features on your map. You will only use pointToLayer for point data.

Part 2: Leaflet events

  1. Start with a Leaflet map (not made with CartoDB.js) from previous work. This might be the one you were working with in Part 1.
  2. Add an event handler to the map for one of the events listed in the docs.
  3. Add an event handler to the features for one of the events listed in the docs for paths. This jsbin might be helpful.

Part 3: Using plugins in Leaflet

  1. Start with a Leaflet map (not made with CartoDB.js) from previous work. This might be the one you were working with in Parts 1 and 2.
  2. Pick a plugin that you would like to add to the map.
  3. Follow the plugin's directions to add the plugin to your map. This will vary by plugin, but generally you will:
    1. Add the plugin's code (JavaScript and often CSS) to your page. This entails either downloading the code into the folder you're working out of, or if you're using a service like JS Bin, using RawGit to create hosted versions of those files.
    2. Invoke the plugin after creating your map.
  4. If you run into trouble, here's a working example using Leaflet.MapPaint. Please use it for guidance but use a different plugin.

Part 4: Using Leaflet plugins through CartoDB.js

  1. Start with a page that loads a CartoDB visualization using CartoDB.js.
  2. Pick a new Leaflet plugin you would like to try.
  3. Add the plugin to your page.
  4. In createVis().done(), use vis.getNativeMap() to get the Leaflet map that underlies the visualization and initialize the plugin.
  5. If you run into trouble, here's a working example using Leaflet.MapPaint with CartoDB.js. Please use it for guidance but use a different plugin.
